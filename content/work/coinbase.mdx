Internal security tooling design — partnered with internal teams to build interfaces for rule analysts to write and test rule logic (fraud/risk detection workflows).

Goals: Make rule authoring and testing fast and legible for analysts. Reduce friction and error in a high-stakes workflow where bad logic has real downstream consequences. Improve confidence in rule output before deployment.

Outcomes: Tooling shipped to support analyst rule-writing and testing. Workflow improvements for an internal user base operating in security-critical conditions.
